# H100 Quickstart — Stage 5 Benchmark Walkthrough

**Goal:** Run the Model B benchmark on a rented H100 with the least possible effort. Total: ~20-30 min wall clock, ~$1-3 in compute cost.

---

## Step 1: Push the repo to GitHub (one-time, 2 min)

From your laptop in `C:\Users\gowri\ratcliff`:

```powershell
# Create a GitHub repo first (web): https://github.com/new
# Name it "ratcliff", visibility: private (or public — either works)
# Don't add README/license/.gitignore — we already have those.

# Then push:
git remote add origin https://github.com/YOUR_USERNAME/ratcliff.git
git branch -M main
git push -u origin main
```

If you'd rather keep it private, GitHub will ask for auth — use a Personal Access Token: https://github.com/settings/tokens (just `repo` scope is enough).

After this, your repo URL is `https://github.com/YOUR_USERNAME/ratcliff.git`. Save it.

## Step 2 — Option A (recommended): Vast.ai (~$1.50/hr, no SSH needed)

### 2A.1 Sign up at vast.ai

Go to https://vast.ai, click Sign Up, fund the account with ~$10 (Stripe accepts cards).

### 2A.2 Rent an H100

1. Click "Templates" → "PyTorch (cuDNN Runtime)"
2. Click "Search" / "Console" → filter:
   - GPU: H100 (any variant)
   - Disk: ≥ 30 GB
   - Sort by: Price (cheapest first)
3. Pick a listing. Click "Rent".

### 2A.3 Paste the one-shot script as On-Start

When renting, find the **"On-start script"** field (or similar — different templates expose this differently). Paste:

```bash
export REPO_URL="https://github.com/YOUR_USERNAME/ratcliff.git"   # ← your repo
bash <(curl -sSL https://raw.githubusercontent.com/YOUR_USERNAME/ratcliff/main/scripts/h100_setup_and_run.sh)
```

If the template doesn't have an on-start field, click "Launch", wait for it to start, then click "Open Web Terminal" (Vast provides one) and paste the same commands.

### 2A.4 Watch the logs

Vast.ai's instance page shows stdout in real time. You'll see:
- `Installing git/python...`
- `pip install` (slowest part, ~3-5 min)
- `JAX devices: [CudaDevice(id=0)]`
- The benchmark sections running (~10-20 min total)
- `DONE. Results saved to /workspace/h100_results.txt`

### 2A.5 Get the results

In the web terminal:
```bash
cat /workspace/h100_results.txt
```

Copy the entire output, paste it back to Claude.

### 2A.6 STOP the instance

Click "Destroy" on the Vast.ai instance page. **This is critical — they bill by the second.**

**Total expected cost:** $1-3 for ~30 min runtime at $1.50-2.50/hr.

## Step 2 — Option B: Lambda Labs (cleanest, $2.50/hr)

### 2B.1 Sign up at lambdalabs.com

Web signup. Fund with card. They auto-allocate.

### 2B.2 Launch an instance

Cloud → On-demand → choose `gpu_1x_h100_pcie` (~$2.50/hr).

You can pick "PyTorch 2.x" as the base image.

### 2B.3 SSH in

They give you `ssh ubuntu@<ip>` after the instance boots.

```bash
export REPO_URL="https://github.com/YOUR_USERNAME/ratcliff.git"
curl -sSL "https://raw.githubusercontent.com/YOUR_USERNAME/ratcliff/main/scripts/h100_setup_and_run.sh" | bash
```

(Lambda doesn't have an on-start field, so you SSH in to paste.)

### 2B.4 Get results

Once it finishes:
```bash
cat /workspace/h100_results.txt
```

Then `Terminate` the instance from Lambda's web console.

## Step 2 — Option C: HF Jobs (one command from your laptop, no rental UI)

**Requires:** Hugging Face Pro account ($9/mo, https://huggingface.co/pro).

If you already have HF Pro:

```powershell
# One-time setup on your laptop
pip install -U huggingface_hub
hf auth login
```

Then submit the job (from your laptop, repo doesn't even need to be cloned anywhere):

```powershell
hf jobs uv run `
    --flavor a100-large `
    --timeout 1h `
    --secrets HF_TOKEN=`$HF_TOKEN `
    "https://raw.githubusercontent.com/YOUR_USERNAME/ratcliff/main/scripts/h100_model_b_benchmark.py"
```

HF will queue the job, run it on their infra, and stream logs back to your terminal. When done, the output is in the job's web UI: https://huggingface.co/jobs

**Note:** A100 isn't H100 (HF Jobs doesn't expose H100 to public yet as of last check). A100 is ~50-70% the throughput of H100 for our workload — still proves the speedup story. If you need true H100, stick with Vast.ai.

## Troubleshooting

### "JAX devices: [CpuDevice(id=0)]" — JAX didn't see the GPU

CUDA install issue. Try forcing the wheel:
```bash
pip install --upgrade "jax[cuda12]==0.4.30" -f https://storage.googleapis.com/jax-releases/jax_cuda_releases.html
```

### Repo clone fails (private repo)

You need a PAT. Either:
- Make the repo public temporarily
- Or use `https://USER:TOKEN@github.com/USER/ratcliff.git` as REPO_URL

### Out of memory at large chunk_size

The benchmark script handles this — it falls back automatically to smaller chunks. You'll see `chunk_size=1024: FAILED` and it'll move on. The reported "best" chunk will be whatever fit.

### `pip install -e .[dev,fit]` fails on `jaxopt`

jaxopt is deprecated. If pip can't find it:
```bash
pip install jaxopt==0.8.5
```

### Benchmark hangs at "Section 4 — fit_simplex_b"

The full simplex fit can take 5-15 min on H100 with 500 iters. Be patient. If it's still going at 30 min, check `nvidia-smi` — if GPU utilization is high, it's working; if zero, something stalled.
